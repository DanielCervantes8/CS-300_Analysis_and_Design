# CS-300_Analysis_and_Design

# Reflection

# In this course, the main problem I was solving was how to take course data from a file and organize it in a way that makes it easy to search and display. The goal was to allow advisors to quickly find a specific course or
 print out all courses in order. To do this, I had to understand how different data structures work and decide which one would be the best fit for the problem.

# To approach this, I compared vectors, hash tables, and binary search trees. I realized that data structures are important because they directly affect how fast and efficiently a program runs. For example, vectors are
 simple but slower when searching, while hash tables are fast but don’t keep data in order. I ended up choosing a binary search tree because it allows both efficient searching and automatic sorting, which fits the
 requirements of the program best.

# One of the biggest challenges I ran into was working with file input and validating the data, especially making sure that prerequisites actually existed in the file. I solved this by using a two-pass approach: first
 collecting all course numbers, then validating prerequisites before storing the courses. I also had some issues with debugging errors and getting the program to read files correctly, but I worked through those by testing
 smaller parts of the code and fixing issues step by step.

# This project helped me better understand how to design programs before writing code. Writing pseudocode first made it easier to organize my logic and avoid major mistakes. It also showed me how important it is to choose
 the right data structure early on instead of trying to fix things later.

# This project improved how I write code by making it more organized, readable, and modular. I used functions to separate tasks like loading data, searching, and printing, which makes the program easier to maintain and
 update. I also learned how to write code that can handle errors and unexpected input, which makes it more reliable.
